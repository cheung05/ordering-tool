請協助我建立一個「公司內部點餐工具」，使用純 JavaScript、HTML5 與 CSS3。
不需要使用 Vue 或 React 等框架，以保持結構簡單易懂。

## 技術需求
1. 身分驗證：使用 GCP Login 進行登入，取得使用者的 Email。
2. 資料庫連結：直接使用 Google Sheets API。不要使用 GAS 或任何第三方代理。用戶也會另外獲得本試算表的授權
3. 權限與流程邏輯：
   - 使用者登入後，需先比對 `Users` 工作表。
   - 若 Email 不在名單內，顯示「未獲授權」。
   - 若在名單內，讀取其「權限」欄位（管理員/一般成員）。

## 試算表結構說明
請根據以下四張工作表來設計邏輯：
[TodayConfig]：欄位為 [今日開放餐廳]。網頁應根據此表決定要顯示哪些餐廳。範例欄位：
```
今日開放餐廳
梁社漢排骨
50嵐
```

[Menu]：欄位為 [餐廳名稱, 品名, 單價, 分類]。範例欄位：
```
餐廳名稱	品名	單價	分類
梁社漢排骨	炸排骨飯	115	主食
梁社漢排骨	滷排骨便當	115	主食
梁社漢排骨	炸雞腿飯	130	主食
梁社漢排骨	秋葵	35	小菜
50嵐	珍珠奶茶(L)	60	飲品
```


[Users]：欄位為 [姓名, Email, 權限]。範例欄位：
```
姓名	Email	權限
管理員 01	admin@example.com	管理員
員工 01	staff01@example.com	一般成員
員工 02	staff02@example.com	一般成員
```


[Orders]：欄位為 [點餐時間, 訂購人 Email, 餐廳名稱, 餐點內容, 金額, 備註]。範例欄位：
```
點餐時間	訂購人 Email	餐廳名稱	餐點內容	金額	備註
2026/01/19 12:05	staff01@example.com	梁社漢排骨	炸排骨飯	115	不要菜脯
2026/01/19 12:10	staff02@example.com	50嵐	珍珠奶茶(L)	60	微糖少冰
```


# 網頁功能要求
1. 登入介面：顯示一個明顯的 Google 登入按鈕，本專案不會使用到後端的 secret key。
2. 身分檢查：
   - 登入成功後，檢查 Email 是否在 Users 表中。
   - 在名單內者，根據其權限顯示對應功能。
3. 設定今日餐點：
   - 從 [Menu] 撈出餐廳名稱的列表
   - 僅有管理員能夠設定今天選用的餐廳（可複選）
   - 將餐廳設定至 [TodayConfig]
4. 點餐介面：
   - 讀取 [TodayConfig] 取得今日餐廳名稱。
   - 從 [Menu] 中過濾出「屬於今日餐廳」的餐點。
   - 每個餐點卡片包含：品名、單價、備註輸入框、點餐按鈕。
5. 送出訂單：
   - 點擊後將資料（含當下時間、Email、餐廳、餐點、金額與備註）直接寫入 Orders 表。
   - 成功後顯示提示訊息。
6. 確認訂單：
	 - 列出今天的餐點列表，並且提供一鍵複製，可以貼到 LINE 或其他對話工具供人確認
7. 管理員專區：
   - 若權限為「管理員」，頁面底部會出現紅色的「清空今日點餐」按鈕。
   - 點擊可彈出確認，確認後，會刪除 Orders 表中除了標題列以外的所有資料。

# 輸出格式要求
請將代碼拆分為以下三個檔案，並加入詳細的繁體中文註解：
1. index.html：包含 HTML 結構與 Google API 腳本載入。
